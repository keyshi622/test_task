	Данная программа соответствует всем требованиям: раз в минуту отправляет запрос в бд, извлекает все данные и записывает их в XML файл заданного формата.
	 После помещает в SOAP оболочку и отправляет на локальный сервер. Программа разработана с помощью .net core, вспомогательных библиотек для установки связи, SQLite3 и редактора db browser для создания базы данных, OpenUI для получения SOAP запроса.
	База данных называется "test.db" и находится в \ConsoleApp1\ConsoleApp1\bin\Debug\net9.0\. 
	Исполняемый файл называется "ConsoleApp1" и находится в \ConsoleApp1\ConsoleApp1\bin\Debug\net9.0\.
	Таблица сохраняется в формате xml в файл "Orders.xml" и находится он в \ConsoleApp1\ConsoleApp1\bin\Debug\net9.0\.
	Логи записываются в файл "log.txt", который находится он в \ConsoleApp1\ConsoleApp1\bin\Debug\net9.0\.
	Для создания локального сервера был создан примитивный WSDL, при создании SOAP проекта нужно указать в качестве файла WSDL файл с именем "OrdersService.wsdl", находящийся в \ConsoleApp1\ConsoleApp1\.
	При создании mockservice нужно указать порт "8080", путь "/mockservice" и хост "localhost".
	В корневой папке есть файл "test-soapui-project" для SoapUI со всеми предустановленными настройками.	
	Инструкция к программе: Запустить mockservice. Запустить файл "ConsoleApp1". в OpenUI открыть response1, проверить полученные данные в левой ячейке. при необходимости добавить в бд новые данные для тестирования.
